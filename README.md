# HFU-Test

Dieses Repository ist ein **Test-Repository** zur Übung grundlegender Arbeitsabläufe mit Git und GitHub.

## Ziel

Ziel ist es, folgende Grundlagen zu erlernen:

- Klonen eines Repositorys
- Zusammenarbeit im Team
- Änderungen committen und pushen
- Verwaltung von Zugriffen (private Repositorys und Collaborators)

## Vorbereitung

1. Klonen Sie dieses Repository auf Ihren lokalen Rechner.
2. Erstellen Sie als Teamlead ein **eigenes privates Repository** auf Basis dieses Repositorys.
3. Fügen Sie Ihre **Teammitglieder als Collaborators** hinzu.
4. Fügen Sie die **Lehrperson als Collaborator** hinzu.
5. Arbeiten Sie ab diesem Zeitpunkt ausschließlich in Ihrem privaten Repository.

## Erste Aufgabe im eigenen privaten Repository

Erstellen Sie eine Datei mit dem Namen:

`team_members.txt`


Jedes Teammitglied soll:

- den eigenen Namen eintragen (eine Zeile pro Person)
- die Änderung committen
- die Änderung in das Repository pushen

## Hinweise

- Achten Sie darauf, dass jedes Teammitglied mindestens einen Commit beiträgt.
- Verwenden Sie aussagekräftige Commit-Nachrichten.
- Stimmen Sie sich im Team ab, um Konflikte zu vermeiden.
